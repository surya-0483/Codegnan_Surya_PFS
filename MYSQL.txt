Frontend
---------
Frontend is the visual, interactive layer of a website or application that users see and interact with directly in their browser or device

Backend
--------
Backend is the server side off a web application it is the engine running behind the scenes. Invisible to user but responsible for all the logic, data processing and communication

API (Application programming interface)
----------------------------------------
An API is a set of rules and protocols that allows two different applications or system to talk each other and exchange data without knowing each others internal code

Storage Areas
--------------
As the part of our applications we required to store our data like customers information, billing information, calls information etc...
to store this data we required storage areas, there are two types of storage areas such as
1. Temporary Storage Areas
---------------------------
these are the memory areas where data will be stored temporarily
example: All JVM memory areas ( like Heap area, stack area, method area) once JVM is shutdown all these memory areas will be cleared automatically
2. Permanent Storage Areas
---------------------------
it is also known as persistent storage areas. Here we can store data permanently
example: File systems, Databases, Data Warehouse, Big Data etc..

File Management System (FMS)
----------------------------
A File Management (FMS) is a software that helps in storing, organizing, retrieving and managing data files in a computer system, it operates directly on the file system provided by our operating system and doesn't support relational data management like Database Management system.
Key functions of a file management system
------------------------------------------
storage management: handles how files are stored on physical storage devices
File Retrieval: provides mechanism to access files using file names or paths
File Organization: allows users to organize files into directions or folders
security and accesses control: offers basic security measures like read/write permission
Backup and recovery: may support backup utilities for data recovery

Drawbacks of File Management System
------------------------------------
1. Data Redundancy and inconsistency: multiple copies of the same data may exist in different files leading storage wastage and inconsistence
2. No Data Integrity: Hard to ensure accuracy and consistency of data across multiple files
3. poor data security
4. difficult data retrieval
5. scalability: not well suited for managing large volumes of data or complex operations
6. no relationship between data

Database Management System (DBMS)
---------------------------------
A database management system is a software designed to store, retrieve, manage and manipulate data efficiently in a structured format. it allows users and applications to interact with data using queries without needing to handle low level details like file storage and memory management

key functions of DBMS
----------------------
Data storage, retrieval and update
user accessed control and security
data integrity management
transaction management and concurrency control
backup recovery

Advantages
-----------
1. reduced data redundancy
2. improved data integrity and consistency
3. enhanced data security
4. better data sharing among users

ACID properties
----------------
Atomicity (A): all or nothing
Consistency (C): the database remains in a valid state
Isolation (I): transactions do not interface with each other but occur simultaneously
Durability (D): committed changes are permanent

data vs database vs DBMS
-------------------------
data: raw facts or figures without context
example: 009,surya,90,50k
field: smallest unit of a data in a database (column/attribute)
example: EmpId,EmpName,SID,sname,smarks,Empsalary etc...
Record: collection of related fields (row/tuple)
example: ('009','surya',90,'50k')
Database: organized collection of related records stored together
example: an employee database containing all employee records
DBMS: software used to manage database (create, delete and update)

Data Models in DBMS
--------------------

1. physical data model
-----------------------
Definition: describes how data was physically stored in storage devices
focus: storage details like portioning and file organization
it is used by data administrators

2. logical data model
----------------------
Definition: describes the structures of data (entities, attributes, relationships)
focus: tables, fields, constraints, relationships
it is used by developers and data base architectures

3. relational data model
-------------------------
definition: data is stored in tables(relations) with rows(tuple) and columns(attributes)
example: student, department

4. hierarchal data model
-------------------------
definition: data is represented as tree structure (parent-child relationships)

5. entity relationship model
-----------------------------
definition: A conceptual model that shows entities and attributes and relationships

6. object data model
---------------------
definition: extends databases to handle objects (like oop) storing attributes and methods

what is SQL
------------
SQL stands for structured query language, it is a standardize programming language specially designed for managing and manipulating relational database. It provides a declarative syntax that allows users to define what data they want retrieve, insert update or delete without specifying exactly how database should execute these operations

Types of SQL commands
---------------------

DDL(Data definition language)
-----------------------------
1. create: by using this command we can create a new database, new tables, views triggers, indexes, stored procedures etc
2. alter: it is used to modify the structure of an existing database object
3. drop: it is used to drop the existing database object or table or views drop permanently
4. truncate: this command is used to delete all rows from table
5. rename: by using command to rename the column names or table names

DML(Data manipulation language)
-------------------------------
1. insert: by using this command to insert some records in a table
2. update: this command is used to modify the existing records in table
3. delete: by using command to delete some records based on certain conditions
4. lock: it can be used to control access to a resource, ensuring that only one transaction can access it at a time preventing conflicts

DCL(Data control language)
--------------------------
1. grant: this command is used to give specific permissions to user in database this permissions might be include the ability to select or delete as well as create or drop tables
2. revoke: the revoke command is used to take away granted permissions from user

TCL(transaction control language)
---------------------------------
1. commit: this command is used to permanently save the changes made during the current transaction once transaction is committed changes become permanent and visible to other transactions
2. rollback: if an error occurs or if the transaction needs to be canceled for any rollback is used to revert the database
             In simple it is used to undo the changes made during the current transactions 
3. save point: it is used to set a point within the transaction which you can later rollback

DQL(Data query language)
------------------------
1. select: by using this command we retrieve data from one or more tables in a database
